# Android Release Keystore — Setup Guide

⚠️ **Security note:** the keystore and its passwords are **permanent and irrecoverable**. If you lose them, you cannot publish updates to your app on Google Play — you'd have to ship as a new app under a different package name. Store them securely (password manager + encrypted backup in cold storage).

---

## Step 1 — Generate the keystore

Open a terminal on your dev machine and run:

```bash
keytool -genkey -v \
  -keystore flyconnect-release.jks \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -alias flyconnect
```

You'll be prompted for:

| Prompt | Example answer |
|--------|----------------|
| Enter keystore password | _strong 16+ char password — save it!_ |
| Re-enter new password | _same as above_ |
| What is your first and last name? | FlyConnect Release |
| What is the name of your organizational unit? | Engineering |
| What is the name of your organization? | AppCurb Technologies |
| What is the name of your City or Locality? | _your city_ |
| What is the name of your State or Province? | _your state_ |
| What is the two-letter country code? | US _(or your ISO code)_ |
| Is CN=..., OU=..., O=... correct? | yes |
| Enter key password for <flyconnect> | _press Enter to reuse keystore password — simpler_ |

Output: `flyconnect-release.jks` in the current directory.

---

## Step 2 — Store it locally

Move the keystore **outside** the project tree so it never gets committed:

```bash
# Example: Windows
mkdir %USERPROFILE%\keystores
move flyconnect-release.jks %USERPROFILE%\keystores\

# macOS/Linux
mkdir -p ~/keystores
mv flyconnect-release.jks ~/keystores/
```

Make sure the project's `.gitignore` includes `**/key.properties` and `**/*.jks` (it should already).

---

## Step 3 — Create `android/key.properties`

This file tells Gradle where the keystore is and what the passwords are. **Never commit it.**

Create `android/key.properties`:

```properties
storeFile=/absolute/path/to/flyconnect-release.jks
storePassword=YOUR_KEYSTORE_PASSWORD
keyAlias=flyconnect
keyPassword=YOUR_KEY_PASSWORD
```

> On Windows, use forward slashes or escaped backslashes in `storeFile`:
> `storeFile=C:/Users/you/keystores/flyconnect-release.jks`

Our `android/app/build.gradle.kts` already reads this file and wires it into the release signing config.

---

## Step 4 — Verify signing works locally

```bash
flutter build appbundle --release
```

If successful, the signed AAB is at `build/app/outputs/bundle/release/app-release.aab`.

Verify the signing with:

```bash
jarsigner -verify -verbose -certs build/app/outputs/bundle/release/app-release.aab
```

You should see `jar verified` — not a warning about the debug key.

---

## Step 5 — Get SHA-1 and SHA-256 fingerprints

Firebase needs these for Google Sign-In and Play App Signing verification:

```bash
keytool -list -v \
  -keystore /path/to/flyconnect-release.jks \
  -alias flyconnect
```

Copy the lines marked `SHA1:` and `SHA256:`. In the **Firebase Console → Project Settings → your Android app → Add fingerprint**, paste each one.

Do the same for the **debug** keystore if you want Google Sign-In to work in debug builds:

```bash
# Debug keystore is auto-generated by Android SDK
keytool -list -v \
  -keystore ~/.android/debug.keystore \
  -alias androiddebugkey \
  -storepass android -keypass android
```

## Step 6 — Store the keystore in CI secrets (GitHub Actions)

Our `.github/workflows/ci.yml` expects these four secrets:

| Secret name | Value |
|-------------|-------|
| `ANDROID_KEYSTORE_BASE64` | Base64-encoded contents of the `.jks` file |
| `ANDROID_KEYSTORE_PASSWORD` | Your keystore password |
| `ANDROID_KEY_ALIAS` | `flyconnect` |
| `ANDROID_KEY_PASSWORD` | Your key password (same as keystore password if you pressed Enter) |

Encode the keystore to base64:

```bash
# macOS
base64 -i flyconnect-release.jks -o flyconnect-release.jks.b64

# Linux
base64 flyconnect-release.jks > flyconnect-release.jks.b64

# Windows PowerShell
certutil -encode flyconnect-release.jks flyconnect-release.jks.b64
```

In GitHub → your repo → **Settings → Secrets and variables → Actions → New repository secret**, create the four secrets above. For `ANDROID_KEYSTORE_BASE64`, paste the contents of the `.b64` file (strip any BEGIN/END headers if `certutil` added them).

Delete the `.b64` file afterward — you don't need it on disk.

---

## Step 7 — Back up the keystore

**This is the step people skip and regret.**

Make **at least two** encrypted backups:

1. Encrypted archive on a cloud drive (1Password, iCloud Keychain, Bitwarden, etc.)
2. Encrypted USB drive in a physical safe

The backup should include:

- `flyconnect-release.jks`
- Keystore password
- Key password (if different)
- Key alias (`flyconnect`)
- SHA-1 and SHA-256 fingerprints (for recovery reference)

---

## Play App Signing (recommended)

When you upload your AAB to Play Console for the first time, Google will offer **Play App Signing**. Opt in — Google generates a separate "app signing key" on their side, and you keep the "upload key" (your keystore) as a signature for what you upload.

Benefits:
- If you lose the upload key, you can reset it via Play Console (not possible without Play App Signing)
- Your app binaries on devices are signed by Google's key
- Enables dynamic delivery and smaller installs

There is no downside for a new app.

## Troubleshooting

**"Failed to read key flyconnect from store"**
Check that `keyAlias` in `key.properties` matches the `-alias` used at generation.

**"Keystore was tampered with, or password was incorrect"**
Your `storePassword` is wrong.

**"Key was created with errors: Warning: The JCEKS..."**
Harmless — you generated a JCEKS keystore but the `jks` extension is fine; Play Console accepts both.

**Play Console rejection: "Your upload key certificate doesn't match"**
You generated a new keystore after a previous upload. The first keystore is now your permanent upload key. If truly lost, enroll in Play App Signing and use the **"reset upload key"** flow.
